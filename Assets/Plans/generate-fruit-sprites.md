# Project Overview
- Game Title: Suika Clone (Unity AI Suika)
- High-Level Concept: A Suika (Watermelon) game clone where players drop fruits to merge them into larger ones, aiming for the final watermelon.
- Players: Single player
- Inspiration / Reference Games: Suika Game (Watermelon Game)
- Tone / Art Direction: Cute Cartoon Style
- Target Platform: Standalone Windows 64
- Screen Orientation / Resolution: Landscape
- Render Pipeline: UniversalRP

# Game Mechanics
## Core Gameplay Loop
1. Player moves a dropper at the top of the screen.
2. Player drops a fruit into a container.
3. Fruits collide and merge if they are of the same type.
4. Merging creates the next level of fruit and awards points.
5. The game ends if fruits overflow the container.

## Controls and Input Methods
- Mouse/Touch: Move dropper horizontally.
- Click/Tap: Drop the current fruit.

# UI
- Game Over screen (not yet implemented).
- Score display.
- Next fruit preview.

# Key Asset & Context
- FruitData (ScriptableObjects): 11 assets (Cherry to Watermelon) containing name, level, sprite, color, radius, score, and next fruit link.
- Fruit Prefabs: 11 prefabs with SpriteRenderer, Rigidbody2D, CircleCollider2D, and Fruit script.
- Sprites: 11 fruit sprites to be generated by Unity AI.

# Implementation Steps
## 1. Asset Generation
- 1.1. Get available AI models for sprite generation using `Unity.AssetGeneration.GetModels`.
- 1.2. Generate the "Watermelon" sprite as a style reference. Prompt: "A cute, vibrant cartoon watermelon, high-quality 2D vector illustration, thick clean outlines, bright colors, game asset, white background." Dependency: Step 1.1.
- 1.3. Generate the remaining 10 fruit sprites (Cherry, Strawberry, Grape, Dekopon, Persimmon, Apple, Pear, Peach, Pineapple, Melon) in parallel using the same style. Dependency: Step 1.2.
- 1.4. Remove backgrounds from all 11 generated sprites using `Unity.AssetGeneration.RemoveSpriteBackground`. Dependency: Step 1.3.

## 2. Project Integration
- 2.1. Update the `sprite` field in each `FruitData` ScriptableObject in `Assets/Data/FruitData/`. Dependency: Step 1.4.
- 2.2. Update the `SpriteRenderer` on each Fruit prefab in `Assets/Prefabs/` to use the new sprites. Dependency: Step 2.1.
- 2.3. Verify that the `Fruit.cs` script correctly scales the prefabs based on the new sprite bounds and the `radius` defined in `FruitData`.

# Verification & Testing
- Manual Check: Inspect each Fruit prefab in the editor to ensure the sprite is correctly assigned and the collider matches the visual size.
- Runtime Test: Play the game and drop each fruit type to verify merging and visual consistency.
- Collision Test: Ensure fruits collide naturally and don't overlap strangely due to sprite bounds.
