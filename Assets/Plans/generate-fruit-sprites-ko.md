# 프로젝트 개요
- 게임 제목: 수박 게임 클론 (Unity AI Suika)
- 핵심 컨셉: 플레이어가 과일을 떨어뜨려 같은 종류끼리 병합해 더 큰 과일을 만들고, 최종적으로 수박을 만드는 게임.
- 플레이어: 싱글 플레이어
- 영감/참고 게임: 수박 게임 (Suika Game)
- 톤/아트 방향: 귀여운 카툰 스타일
- 대상 플랫폼: Windows (Standalone 64-bit)
- 화면 방향/해상도: 가로 모드
- 렌더 파이프라인: UniversalRP (URP)

# 게임 메커니즘
## 핵심 게임플레이 루프
1. 플레이어가 화면 상단에서 투하기를 좌우로 움직입니다.
2. 플레이어가 현재 과일을 상자 안으로 떨어뜨립니다.
3. 과일들이 서로 충돌하며, 같은 종류의 과일끼리 닿으면 병합됩니다.
4. 병합 시 다음 단계의 과일이 생성되고 점수를 획득합니다.
5. 과일이 상자 밖으로 넘치면 게임이 종료됩니다.

## 컨트롤 및 입력 방식
- 마우스/터치: 투하기 가로 이동.
- 클릭/탭: 과일 투하.

# UI
- 점수 표시.
- 다음 과일 미리보기.

# 주요 에셋 및 컨텍스트
- FruitData (ScriptableObjects): 11개의 에셋 (체리부터 수박까지). 레벨, 스프라이트, 색상, 반지름, 점수, 다음 과일 링크 포함.
- 과일 프리팹 (Fruit Prefabs): SpriteRenderer, Rigidbody2D, CircleCollider2D, Fruit 스크립트가 포함된 11개의 프리팹.
- 스프라이트 (Sprites): Unity AI로 생성할 11종의 과일 이미지.

# 구현 단계
## 1. 에셋 생성
- 1.1. `Unity.AssetGeneration.GetModels`를 사용하여 스프라이트 생성에 적합한 AI 모델 목록 확인.
- 1.2. 스타일 기준이 될 "수박" 스프라이트 생성. 프롬프트: "귀엽고 활기찬 카툰 수박, 고품질 2D 벡터 일러스트레이션, 굵고 깨끗한 외곽선, 밝은 색상, 게임 에셋, 흰색 배경." (의존성: 1.1단계)
- 1.3. 나머지 10종의 과일(체리, 딸기, 포도, 한라봉, 감, 사과, 배, 복숭아, 파인애플, 멜론)을 동일한 스타일로 병렬 생성. (의존성: 1.2단계)
- 1.4. `Unity.AssetGeneration.RemoveSpriteBackground`를 사용하여 생성된 모든 11종 스프라이트의 배경 제거. (의존성: 1.3단계)

## 2. 프로젝트 통합
- 2.1. `Assets/Data/FruitData/` 경로의 각 FruitData 에셋의 `sprite` 필드 업데이트. (의존성: 1.4단계)
- 2.2. `Assets/Prefabs/` 경로의 각 과일 프리팹의 SpriteRenderer에 새 스프라이트 할당. (의존성: 2.1단계)
- 2.3. `Fruit.cs` 스크립트가 새 스프라이트 크기와 FruitData의 `radius`를 기준으로 프리팹 스케일을 올바르게 조정하는지 확인.

# 검증 및 테스트
- 수동 확인: 에디터에서 각 과일 프리팹을 확인하여 스프라이트가 잘 적용되었는지, 콜라이더가 시각적 크기와 일치하는지 확인.
- 런타임 테스트: 게임을 실행하여 각 과일을 떨어뜨려 보고 병합 및 시각적 일관성 확인.
- 충돌 테스트: 스프라이트 경계로 인해 과일들이 부자연스럽게 겹치거나 튕기지 않는지 확인.
